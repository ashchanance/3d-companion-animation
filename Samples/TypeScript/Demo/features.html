Features

What makes Haruka different.

Not a chatbot. A companion with real runtime, wallet-aware context, and developer-grade infrastructure behind her.

Live

Live2D Real-Time Rendering

Haruka is rendered in real-time using the Live2D Cubism engine. Her expressions are not pre-animated loops - they are generated dynamically from your input, your voice, and the flow of conversation.

The physics engine runs at up to 60 FPS, calculating hair, ribbon movement, and fabric motion directly inside the browser.

Live

Real-time Voice

Talk to Haruka and she talks back. Voice input is captured through the microphone, transcribed in real-time, and turned back into audio as her reply.

The full round-trip from mic input to voice output happens in under 2 seconds.

Live

Open Souls Soul Engine

Haruka is built on the Open Souls Soul Engine. She uses WorkingMemory, MentalProcesses, and cognitiveSteps so she can think before she speaks and stay consistent across sessions.

She remembers you, stays in character, and keeps a more stable response posture over time.

Live

20+ LLM Providers

Haruka is not locked to one AI model. Browser Local, OpenAI, OpenAI Compatible, Claude, DeepSeek, Gemini, Ollama, Aliyun NLS, Web Speech API, Comet API, and more all run through the same provider layer.

Switch anytime from the settings panel without breaking the companion experience.

Live

API Server

Haruka is accessible via API. Any developer, any app, any platform can send a message and receive text, emotion, and memory context back.

POST /chat turns Haruka from a website into a service.

Live

Embed Widget

One line of code and Haruka shows up on another website with her Live2D character, voice, memory, and chat interface.

The current brief points to Atelier as the first live external integration.
